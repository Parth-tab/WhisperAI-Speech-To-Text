# WhisperAI Security & Privacy

WhisperAI is built on a foundation of absolute privacy and local data sovereignty. This document outlines our security posture, data handling practices, and cryptographic integrity measures.

## 1. Guaranteed Data Sovereignty

The core value proposition of WhisperAI is that it works entirely offline.
*   **Zero Data Exfiltration**: Absolutely zero raw audio, transcribed text, or active window context leaves your local Windows machine. 
*   **No Cloud APIs**: We do not use third-party APIs for transcription or intent processing. The LLM (`llama-cpp-python`) and ASR (`faster-whisper`) engines run strictly on local hardware.
*   **Local Telemetry**: Any performance telemetry or logs generated by the application (e.g., RAM usage, latency) are written locally to `%USERPROFILE%\.whisperai\logs\` and are never transmitted automatically.

## 2. Cryptographic Integrity and Model Checksumming

To ensure the AI models haven't been tampered with or poisoned, we employ cryptographic verification.
*   **Model Weights Verification**: Upon downloading or loading GGUF and Whisper model weights, the application verifies their SHA-256 checksums against a known-good ledger.
*   **Binary Integrity**: Our Continuous Deployment (CD) pipeline automatically generates and publishes SHA-256 hashes for all compiled binaries and installer packages.
*   **Code Signing**: Releases are cryptographically signed using valid certificates to satisfy Windows SmartScreen and ensure binary authenticity.

## 3. Vulnerability Reporting

If you discover a security vulnerability or a potential model bypass that could compromise the local isolation of WhisperAI, please report it via GitHub Security Advisories.
